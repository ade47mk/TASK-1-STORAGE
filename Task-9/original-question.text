TASK 9: CoreDNS Troubleshooting

Scenario:
DNS lookups are failing in the Kubernetes cluster. Pods cannot resolve external DNS names (e.g., google.com), and applications are unable to connect to external services. The CoreDNS service needs to be investigated and repaired.

Symptoms:
- External DNS resolution fails from pods
- CoreDNS pods appear to be running but not functioning correctly
- Applications fail to connect to external APIs

Objective:
Investigate why DNS lookups are failing and restore CoreDNS functionality so that both internal cluster DNS and external DNS resolution work correctly.

Requirements:
1. CoreDNS pods must be running and healthy in kube-system namespace
2. External DNS resolution must work (e.g., google.com, github.com)
3. Internal cluster DNS must work (e.g., kubernetes.default)
4. CoreDNS ConfigMap must be correctly configured with valid upstream DNS servers

Tasks:
1. Check CoreDNS pod status and logs
2. Examine CoreDNS ConfigMap configuration
3. Identify the misconfiguration (invalid upstream DNS servers)
4. Fix the CoreDNS ConfigMap
5. Restart CoreDNS pods to apply changes
6. Verify DNS resolution works for both internal and external domains

Common Issues:
- Invalid upstream DNS servers in the 'forward' directive
- CoreDNS ConfigMap misconfiguration
- Need to restart CoreDNS pods after configuration changes

Hints:
- CoreDNS configuration is stored in configmap/coredns in kube-system namespace
- Use 'kubectl edit configmap coredns -n kube-system' to fix configuration
- Valid upstream DNS options: /etc/resolv.conf, 8.8.8.8, 1.1.1.1
- Restart CoreDNS: kubectl delete pods -n kube-system -l k8s-app=kube-dns
- Test DNS: kubectl exec dns-test -- nslookup google.com

Difficulty: Hard
Points: 20
Time Estimate: 15-20 minutes
